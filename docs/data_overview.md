## 四、用户特征（user_feature）

- 每条样本 13~54 个特征，共 **57 个不同 feature_id**
- 包含多种类型：`int_value`、`int_array`、`float_array`、`int_array_and_float_array`

### 按类型分类

| value_type | feature_ids | 说明 |
|------------|-------------|------|
| **int_value** | 1, 3, 4, 50-66, 76, 80, 82, 86-105 | 稀疏类别特征（如性别、年龄分桶等） |
| **int_array** | 5, 18, 53, 54, 67, 74 | 多值类别特征 |
| **float_array** | 68, 81 | 稠密嵌入向量（如用户 Embedding） |
| **int_array_and_float_array** | 69, 70, 71, 72, 73, 83, 84, 85 | 混合类型（类别 ID + 对应权重） |

### 混合类型示例 (feature_id=69)

```python
{
  "int_array":   [4, 2, 3, 5],           # 类别 ID
  "float_array": [60298., 25804., 3220., ...]  # 对应权重/频次
}
```

## 五、序列特征（seq_feature）

seq_feature 是一个 dict，包含 **3 种用户行为序列**：

| 序列名 | 子特征数 | 长度范围 | 平均长度 | 中位数 | 含义 |
|--------|:--------:|:--------:|:--------:|:------:|------|
| **action_seq** | 10 | 1 ~ 3,998 | 2,525 | 2,798 | 广告行为序列（曝光/点击/转化） |
| **content_seq** | 9 | 4 ~ 1,999 | 762 | 587 | 内容消费序列 |
| **item_seq** | 12 | 1 ~ 3,999 | 510 | 364 | 物品交互序列 |

---

### 5.1 action_seq 详解

`action_seq` 包含 **10 个并行对齐的 int_array**，每个 array 描述同一组行为事件的不同属性。所有 10 个 feature 的序列长度 **完全一致**（逐位置对齐），按时间 **降序** 排列（最新行为在最前面）。

#### 对齐关系示意

```
事件 0:  fid19[0], fid20[0], ..., fid28[0]  → 同一个行为事件的 10 个属性
事件 1:  fid19[1], fid20[1], ..., fid28[1]  → 另一个行为事件
...
```

#### 逐事件示例（样本 0，前 5 个事件）

```

-----|------|------|------|-------|---------|------|------|------------

   1 |    1 |  201 |  890 |     0 |   2882  |   10 |    9 | 02-08 17:50

```

## 六、特征值类型说明

所有特征使用统一的 struct schema，根据 `feature_value_type` 决定取哪个值字段：

| feature_value_type | 填充字段 | 说明 |
|:------------------:|:--------:|------|
| `int_value` | `int_value` | 单个整数，稀疏类别 ID |
| `float_value` | `float_value` | 单个浮点数 |
| `int_array` | `int_array` | 整数数组（多值类别或序列） |
| `float_array` | `float_array` | 浮点数组（稠密 embedding） |
| `int_array_and_float_array` | `int_array` + `float_array` | 混合类型（类别 ID + 对应权重） |

## 七、竞赛建模要点

```
┌───────────────────────────────────────────────────────────┐
│  输入                                                      │
│  ├─ user_feature (57 个特征, 含稀疏/稠密/混合)              │
│  ├─ item_feature (15 个特征, 主要稀疏)                     │
│  └─ seq_feature                                           │
│      ├─ action_seq  (10 特征 × 最长 ~4000 步)             │
│      ├─ content_seq ( 9 特征 × 最长 ~2000 步)             │
│      └─ item_seq   (12 特征 × 最长 ~4000 步)              │
│                                                            │
│  目标: 预测 CVR (action_type=2 即转化)                      │
│  指标: AUC of ROC                                          │
│  约束: 推理延迟限制                                         │
└───────────────────────────────────────────────────────────┘
```

### 关键挑战

1. **序列极长** — action_seq 中位数 2,798 步，最长 3,998 步，需要高效的长序列建模
2. **特征异构** — 稀疏 ID、稠密向量、混合权重类型并存，需要统一 tokenization
3. **正样本稀疏** — 转化率仅 ~10%（样本级）/ 0.2%（行为级），类别不平衡
4. **推理延迟约束** — 不能使用过重的模型，需要在效果和效率间权衡
