# 腾讯广告算法大赛 — KDD Cup 2026

> **Towards Unifying Sequence Modeling and Feature Interaction for Large-scale Recommendation**

## 🎯 赛题概述

设计一种 **统一的 tokenization 方案** 和 **同质可堆叠的 backbone**，在单一架构中同时建模：
- **序列用户行为**（曝光、点击、转化等时序事件）
- **非序列多域特征**（用户属性、物品类别、上下文等）

目标：预测 **CVR（转化率）**。

### 背景
推荐系统两大范式——**特征交叉模型** 和 **序列模型**——长期独立发展，导致工业系统架构碎片化、交叉融合浅层化。比赛鼓励参赛者打破这一隔阂，构建统一 backbone。

---

## 📊 评分规则

| 指标 | 说明 |
|------|------|
| **主指标** | **AUC of ROC**（越高越好） |
| **延迟门控** | 每次提交必须满足 **赛道 + 轮次特定的推理延迟上限**，超时提交直接判定无效 |
| **排名方式** | 初赛使用公开测试集（Leaderboard），决赛使用私有测试集 |
| **Tie-breaking** | AUC 相同 → 推理延迟更低者胜出 → 仍相同则提交时间更早者胜出 |

> [!IMPORTANT]
> 即使 AUC 分数极高，**推理延迟超出预算的提交将被标记为无效**，不参与排名。

## 📂 数据集

来自腾讯广告平台的 **大规模工业数据集**，包含两部分：

1. **用户行为序列** — 用户与物品的交互事件（曝光、点击、转化），带时间戳和行为类型
2. **非序列多域特征**：
   - 用户特征（婚姻状况、性别、年龄等）
   - 物品特征（类型、品类、广告主类型等）
   - 上下文特征（设备品牌、操作系统类型等）
   - 交叉特征（用户 Embedding 等）

> 稀疏特征为匿名化整数 ID，稠密特征为定长浮点向量。数据格式为 JSON。
